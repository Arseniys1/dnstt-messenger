## Назначение проекта

DNSTT Messenger предназначен для ситуаций, когда обычные каналы связи блокируются DPI, фильтрацией DNS или ограничениями провайдера.  
Транспорт через DNS позволяет держать связь там, где HTTPS/VPN может быть нестабильным или полностью запрещенным.

## Основные возможности

- End-to-end шифрование сообщений между клиентами.
- Транспорт сообщений поверх DNS-туннеля.
- Клиенты для Android и Electron (desktop).
- Мультиязычный интерфейс (по умолчанию English, с переключением в настройках).
- Логи приложения на английском для унифицированной диагностики.

## Часть A: Настройка сервера

### 1) Подготовьте инфраструктуру

Минимально нужен:

- Linux-сервер с публичным IP.
- Домен и доступ к управлению DNS-зоной.
- Открытые UDP/TCP порты в firewall (в зависимости от вашей схемы развёртывания).

Рекомендуется:

- Выделенный VPS (не shared hosting).
- Отдельный поддомен под туннель, например `t.example.com`.
- Мониторинг доступности сервера и DNS.

### 2) Настройте DNS-записи

- Создайте записи для поддомена туннеля согласно конфигурации backend.
- Проверьте, что авторитативные NS отвечают стабильно из внешней сети.
- Убедитесь, что TTL не слишком высокий для этапа внедрения.

Проверка:

- `dig NS example.com`
- `dig t.example.com`

### 3) Сконфигурируйте и запустите backend

Серверная часть должна быть настроена на:

- bind-адрес и порт,
- домен/поддомен туннеля,
- ключи и параметры шифрования,
- политики лимитов и таймаутов.

Перед запуском в production:

- включите ротацию логов,
- задайте автозапуск через systemd/supervisor,
- проверьте восстановление после рестарта.

### 4) Проверка доступности сервера

- Поднимите сервер и убедитесь, что порт слушается.
- Выполните тестовое подключение клиента.
- Проверьте server logs на ошибки рукопожатия/маршрутизации.

## Часть B: Настройка клиента

### 1) Android клиент

Базовые шаги:

1. Установите APK (или debug/release сборку).
2. Откройте экран подключения и укажите параметры сервера.
3. Подключитесь и дождитесь успешного статуса сессии.
4. Перейдите в Settings и выберите язык интерфейса из списка.

Важно:

- Смена языка должна применяться без выхода с сервера (in-session settings).
- Если видите поврежденные символы, проверьте локаль устройства и кодировку ресурсов.

### 2) Electron клиент

Базовые шаги:

1. Запустите приложение.
2. Введите параметры подключения и откройте сессию.
3. Перейдите в экран Settings (не в раздел global chat).
4. Выберите язык и проверьте, что UI обновился корректно.

Важно:

- Селектор языка должен быть стилистически согласован с темой приложения.
- Тексты настроек не должны отображаться внутри канала чата.

### 3) Локализация и логирование

- UI-строки берутся из i18n-словарей/ресурсов.
- Ключи переводов стабильны между платформами.
- Служебные логи оставляются на английском языке для поддержки и triage.

## Часть C: Сборка из исходников

### Требования

- `git`
- `Go` (версия согласно проекту)
- `Node.js` + пакетный менеджер для Electron-части
- Android toolchain (`Android Studio`, `SDK`, `Gradle`) для мобильного клиента

### Android

1. Откройте Android-проект в Android Studio.
2. Синхронизируйте Gradle-зависимости.
3. Соберите `debug` или `release` вариант.
4. Убедитесь, что все `strings.xml` и локализованные ресурсы в UTF-8.

### Electron

1. Установите зависимости: `npm install` (или ваш пакетный менеджер).
2. Запустите dev-режим: `npm run dev`.
3. Сборка релиза: `npm run build`.
4. Проверьте, что i18n-файлы включены в итоговый пакет.

## Диагностика и частые проблемы

### Клиент не подключается

- Неверные DNS-записи или недоступен авторитативный DNS.
- Блокировка UDP/портов у провайдера или на сервере.
- Ошибка в параметрах endpoint/ключей.

### Неправильные символы в UI (mojibake)

- Файл перевода сохранен не в UTF-8.
- Android ресурсы собраны с поврежденной кодировкой.
- Неверная локаль в fallback-цепочке.

### Переключение языка не применилось

- Проверьте, что изменение языка обновляет state в runtime.
- Убедитесь, что кэш переводов сбрасывается/перерисовывается UI.
- Проверьте наличие ключей в выбранной локали.

### Настройки показываются не в том экране

- Для Electron экран настроек должен быть отдельным view/state.
- Маршрутизация не должна монтировать Settings в канал Global Chat.

## Рекомендации по безопасности

- Используйте отдельный домен/поддомен только для мессенджера.
- Регулярно ротируйте ключи и секреты.
- Ограничьте SSH-доступ к серверу (ключи, fail2ban, firewall).
- Включите бэкапы конфигурации и recovery-процедуры.
- Минимизируйте данные в логах, не пишите чувствительные payload.

### Операционная безопасность для стран с цензурой

- Не публикуйте публично адреса рабочих серверов.
- Разделяйте тестовые и production-инстансы.
- Используйте нейтральные naming conventions для инфраструктуры.
- Планируйте резервные домены/каналы на случай блокировок.
